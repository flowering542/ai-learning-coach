# AI 学习教练平台 - PRD

## 1. 产品概述

### 1.1 产品定位

**不是考试工具，是「学习教练」**

| 维度 | 传统培训 | AI 教练 |
|------|---------|---------|
| 核心 | 灌输知识 | 引导思考 |
| 关系 | 单向输出 | 对话陪伴 |
| 目标 | 考过就行 | 真正掌握 |
| 情感 | 孤独学习 | 被关注、被鼓励 |
| 结果 | 考完就忘 | 内化为能力 |

### 1.2 一句话描述

用 AI 规模化地提供「被关注、被理解、被鼓励」的个性化学习体验

### 1.3 目标用户

- **主要用户**：医疗从业者备考职称（痛点：工作 busy、知识点零散）
- **次要用户**：其他职业资格考试考生（痛点：孤独学习、容易放弃）

### 1.4 核心价值主张

**AI 会的，自己会还是不一样的**

我们提供的不是答案，而是：
- ✅ 记住你的薄弱点，针对性出题
- ✅ 进步时及时肯定（"比上周多对 3 题！"）
- ✅ 沮丧时鼓励（"80% 的人都在这里卡过"）
- ✅ 考前焦虑时陪伴（"你已经准备了 87 天"）
- ✅ 考过后庆祝（"这是你应得的"）

## 3. 产品功能

### 3.1 学生端（QQ/微信）

1. **智能对话学习**
   - 碎片化知识点讲解
   - 苏格拉底式提问
   - 错题归因

2. **个性化学习路径**
   - 入学测评
   - 动态调整
   - 每日任务

3. **情感陪伴系统**
   - 进度可视化
   - 成就徽章
   - 考前疏导

4. **题库练习**
   - 章节练习
   - 模拟考试
   - 错题本

### 3.2 管理端（Web 仪表盘）

5. **学员管理**
   - 列表、激活码、权限

6. **数据看板**
   - 学员数、活跃度、进度热力图

7. **内容管理**
   - 课程编辑、题库管理

### 3.3 用户认证流程

```
学生 QQ: "激活 XH2025ABC"
    ↓
验证激活码 → 绑定 QQ → 更新 auth.json
    ↓
回复："✅ 激活成功！有效期至 2026-12-31"
```

## 4. 技术架构

### 4.1 核心组件

- 消息路由（deliver 模块）
- 会话管理（session-key 隔离）
- 工具调用（skills）
- Cron 定时任务

### 4.2 会话隔离策略

配置：`dmScope = "per-channel-peer"`

```
学生 A: agent:course:qqbot:dm:12345
学生 B: agent:course:qqbot:dm:67890
```

### 4.3 技术栈

| 层级 | 技术 |
|------|------|
| 运行时 | Node.js 22+ |
| 框架 | OpenClaw |
| 渠道 | QQ Bot |
| 管理端 | Next.js + shadcn/ui |
| 数据 | SQLite + JSONL |
| 部署 | PM2 + 云服务器 |
